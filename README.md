# Мой договор · Mon contrat

Тренажёр привычки **«сначала цена — потом работа»**. Трёхъязычный (РУ / FR / НХ). Автор: **Djeirkhanov**.

**Два режима работы:**
- **Локальный** (без настройки) — данные в телефоне пользователя, офлайн, без входа.
- **Облачный** (с Supabase) — вход через Google, данные в облаке, доступ с любого телефона, приватность через RLS. Настройка — см. **`SETUP.md`**.

Приложение само определяет режим: если заданы переменные `VITE_SUPABASE_*` — включается вход и облако; если нет — работает локально.

## Запуск

```bash
npm install      # установить зависимости (Vite)
npm run dev      # локальный сервер разработки (обычно http://localhost:5173)
npm run build    # сборка в dist/index.html (ОДИН самодостаточный файл)
npm run preview  # посмотреть собранную версию
```

После `npm run build` получается **`dist/index.html`** — единый файл со всем CSS и JS внутри. Его можно:
- открыть напрямую на телефоне (работает офлайн);
- опубликовать как артефакт;
- отправить одним файлом.

## Архитектура

```
mon-contrat/
├── index.html              # разметка (header, экраны, нижняя навигация, онбординг)
├── vite.config.js          # сборка в один файл (vite-plugin-singlefile)
├── package.json
└── src/
    ├── main.js             # точка входа: подключает стили и логику
    ├── styles/
    │   ├── tokens.css       # дизайн-токены и темы (светлая/тёмная)
    │   └── components.css    # стили всех компонентов
    ├── core/
    │   ├── store.js         # localStorage-кэш + отправка изменений в облако
    │   ├── supabase.js      # клиент Supabase (создаётся только при заданных env)
    │   ├── auth.js          # вход/выход через Google
    │   └── cloud.js         # синхронизация: hydrate (облако→кэш) + очередь отправки
    ├── i18n/
    │   ├── ru.js            # словарь (данные) — русский
    │   ├── fr.js            # словарь — французский
    │   ├── ce.js            # словарь — чеченский
    │   └── index.js         # t(), текущий язык, applyStatic(), setLang()
    └── app.js              # вся логика: профиль, навигация, история (кнопка «назад»),
                            # экран «Сегодня», ритуал, дневник/архив, прайс,
                            # программа 90 дней, месячный аудит
```

### Как это устроено
- **Данные и переводы** отделены от логики: `src/i18n/{ru,fr,ce}.js` — чистые словари, `src/i18n/index.js` — механизм перевода.
- **Стили** разбиты на токены (переменные, темы) и компоненты.
- **Логика** (`app.js`) импортирует утилиты из `core` и переводы из `i18n`.
- Разметка использует инлайн-`onclick`, поэтому обработчики экспортируются в `window` (см. блок `Object.assign(window, …)` в конце `app.js`).
- Смена языка идёт через `setLang()` в `i18n`, который дёргает колбэк `onLangChange`, зарегистрированный в `app.js`, — и приложение перерисовывается.

### Чеченский язык
Чеченские тексты (`src/i18n/ce.js`) — **черновик**, требующий проверки носителем языка. Править можно прямо в файле, не трогая остальной код.

## Дальнейшее развитие
`app.js` можно при желании разбить на `src/features/{today,ritual,prices,log,program,audit,profile}.js` — структура к этому готова (модули core/i18n уже выделены).
